# Additional information

## Category model 

The attachment model contains all the information about the category of the courses.

### Properties

<Properties>
  <Property name="id" type="string">
    Unique identifier for the attachment.
  </Property>
  <Property name="name" type="string">
    The name of the Category.
  </Property>
</Properties>

## Role model

The attachment model contains all the information about the role type of the user. Decide whether the user is a student or facilitator.

### Properties

<Properties>
  <Property name="id" type="string">
    Unique identifier for the attachment.
  </Property>
  <Property name="name" type="string">
    The name of the role type.
  </Property>
</Properties>

## List all category {{ tag: 'GET', label: '/v1/categories' }}

<Row>
  <Col>

    This endpoint allows you to retrieve a list of all of the categories.
  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="GET" label="/v1/categories">

    ```bash {{ title: 'cURL' }}
    curl -G https://api.protocol.chat/v1/categories \
    ```

    ```js
    import ApiClient from '@example/protocol-api'

    const client = new ApiClient(token)

    await client.categories.list()
    ```

    ```python
    from protocol_api import ApiClient

    client = ApiClient(token)

    client.categories.list()
    ```

    ```php
    $client = new \Protocol\ApiClient($token);

    $client->categories->list();
    ```

    </CodeGroup>
