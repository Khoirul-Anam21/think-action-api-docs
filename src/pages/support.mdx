# supports

supports is like a follower. in this model only supporting for creation, since if we filter by supporting user it will be the user supporter

## The supporting model

The support model contains all the information about supports.

### Properties

<Properties>
  <Property name="id" type="string">
    Unique identifier for the Supporter.
  </Property>
  <Property name="user_id" type="string">
    Identifier of the current user
  </Property>
  <Property name="supporting" type="object">
    Identifier of the supported by current user
  </Property>
</Properties>

## Add new supporting user  {{ tag: 'POST', label: '/v1/supports' }}

<Row>
  <Col>

    This endpoint allows you to create a new user to be supported, because the current user can't add its own supporter
    
    ### Required attributes

    <Properties>
      <Property name="supporting_id" type="string">
        Unique identifier of user who want to be supported
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/v1/supports">

    ```js
    const response = await axios.post('/v1/supports', {
      supporting_id: 'user123',
    });
    ```

    </CodeGroup>

    ```json {{ title: 'Response' }}
     {
      "id": "fadfGas45",
      "acknowledged": true
     }
    ```

  </Col>
</Row>
