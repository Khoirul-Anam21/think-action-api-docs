# supports

supports are where the facilitator announce all the important information to its student. rather than in the comment, supports should have a special and noticeable place {{ className: 'lead' }}

## The supporter model

The support model contains all the information about your supports.

### Properties

<Properties>
  <Property name="id" type="string">
    Unique identifier for the Supporter.
  </Property>
  <Property name="user_id" type="string">
    Identifier of the User associated with the Supporter.
  </Property>
  <Property name="supporterUser_id" type="string">
    Identifier of the User who is supporting.
  </Property>
</Properties>

## Add new supporting user  {{ tag: 'POST', label: '/v1/supports' }}

<Row>
  <Col>

    This endpoint allows you to create a new user to be supported, because the current user can't add its own supporter
    
    ### Required attributes

    <Properties>
      <Property name="user_id" type="string">
        Unique identifier of user supported first.
      </Property>
      <Property name="course_id" type="string">
        Unique identifier of the course this support belongs to.
      </Property>
      <Property name="description" type="string">
        The content of the support.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/v1/supports">

    ```js
    const response = await axios.post('/v1/supports', {
      user_id: 'user123',
    });
    ```

    </CodeGroup>

    ```json {{ title: 'Response' }}
     {
      "id": "fadfGas45",
      "acknowledged": true
     }
    ```

  </Col>
</Row>
